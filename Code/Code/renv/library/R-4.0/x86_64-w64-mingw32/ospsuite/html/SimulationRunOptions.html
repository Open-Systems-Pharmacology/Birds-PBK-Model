SimulationRunOptions {ospsuite}R Documentation

SimulationRunOptions

Description

Options to be passed to the simulation engine

Super classes

ospsuite::Printable -> ospsuite::DotNetWrapper -> SimulationRunOptions

Active bindings

numberOfCores

(Maximal) number of cores to be used. This is only relevant when simulating a population simulation. Default is getOSPSuiteSetting("numberOfCores").

checkForNegativeValues

Specifies whether negative values check is on or off. Default is TRUE

showProgress

Specifies whether progress bar should be shown during simulation run. Default is getOSPSuiteSetting("showProgress")
